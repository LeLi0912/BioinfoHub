# Brainstorming

Guide the user through structured ideation. Adapt the approach to the problem type.

## Modes

### Divergent (generate options)
- Generate 5-10 distinct options, each with a one-line summary
- Push beyond the first obvious answers — include at least 2 unconventional options
- For each option: brief pros/cons, feasibility, and what makes it interesting
- Don't evaluate yet — just list

### Convergent (narrow down)
- Group related ideas into themes
- Score against criteria the user cares about (simplicity, speed, maintainability, etc.)
- Recommend top 2-3 with rationale
- Note which ideas combine well

### Structured prompts
When the user is stuck, offer framing lenses:
- "What would the simplest version look like?"
- "What would you build if you had 10x the time?"
- "What would [well-known project] do here?"
- "What's the version that would take 1 day? 1 week? 1 month?"

## Output
- Always end with actionable next step: a decision to make, a prototype to build, or a question to research
- If the user lands on a direction, offer to transition into planning or implementation
