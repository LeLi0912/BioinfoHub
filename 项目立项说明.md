# BioinfoHub — 项目立项说明

---

## 一、项目概述

| 项 | 内容 |
|---|------|
| 项目名称 | **BioinfoHub**（暂定） |
| 项目类型 | Web 应用平台 |
| 技术栈 | FastAPI + React 18 + MySQL 8.0 |
| 一句话定位 | 面向中文生物信息学研究人员的现代工具发现与追踪平台 |

---

## 二、项目背景与用户痛点

### 2.1 行业现状

生物信息学工具呈爆发式增长。据 bio.tools 统计，全球已有超过 **33,000** 个生信软件工具。平均每个工作日有 3-5 个新工具发布。

### 2.2 核心痛点

| 痛点 | 量化描述 |
|------|----------|
| 发现困难 | 新工具散落在 GitHub、bioRxiv、PubMed 等多处，无统一入口 |
| 信息碎片化 | 同一工具的信息分散在论文、仓库、文档中 |
| 选择成本高 | 同类工具众多（如 100+ 变异检测工具），无并排对比 |
| 时效性差 | 现有平台更新周期长（bio.tools 依赖社区提交，延迟数周到数月） |
| 中文生态缺失 | 国际主流平台无中文支持，国内用户门槛高 |

### 2.3 竞品对比

| 维度 | bio.tools | BioCode | Awesome-Bioinfo | **本项目** |
|------|-----------|---------|-----------------|-----------|
| 工具规模 | ~33,000 | ~7,300 | ~500 | MVP 1,000+ |
| 前端技术 | Django 模板 | 传统 Web | Markdown 静态 | **React SPA** |
| 数据获取 | 社区提交 | 文献爬取+提交 | 手动维护 | **多源自动爬取** |
| 中文支持 | 无 | 无 | 无 | **全中文界面** |
| 工具对比 | 无 | 无 | 无 | **并排对比（≤4）** |
| 趋势分析 | 无 | 无 | 无 | **涨星排行+分类趋势** |
| 开放 API | 有 | 无 | 无 | **REST API + OpenAPI 文档** |

## 三、产品范围

### 3.1 MVP（V1.0）：本期交付

| 功能模块 | 核心能力 |
|----------|----------|
| 工具列表 | 分页展示、关键词搜索、分类/语言筛选、多维度排序 |
| 工具详情 | 完整元数据、GitHub 统计、论文链接、README 摘要 |
| 工具对比 | 最多 4 个工具并排对比 Stars/Forks/Language/License/活跃度 |
| 趋势面板 | 涨星最快 Top 10、分类分布饼图、语言分布柱状图、新增趋势折线图 |
| 数据同步 | GitHub API + bio.tools API 自动聚合，每日定时同步 + 手动触发 |
| REST API | 10 个端点，OpenAPI 自动文档 |

### 3.2 明确不做（Out of Scope）

- 用户系统（注册/登录/收藏）
- 邮件/Webhook 订阅
- NLP 自动标签抽取
- 用户投稿/审核
- 工具评分与评论
- 智能推荐

## 四、量化验收标准

### 4.1 数据指标

| 验收项 | 量化标准 | 测量方法 |
|--------|----------|----------|
| 种子工具数量 | ≥ 500 条 | `SELECT COUNT(*) FROM tools` |
| 含完整元数据比例 | ≥ 80% 工具有 stars/language/license | `SELECT COUNT(*) FROM tools WHERE github_stars > 0 AND github_language IS NOT NULL AND github_license IS NOT NULL` |
| 数据源覆盖 | GitHub + bio.tools 两个源均有数据入库 | 查 `sync_logs` 表，两个 source 均有 success 记录 |
| 分类覆盖率 | ≥ 90% 工具至少归属一个分类 | `SELECT COUNT(*) FROM tools t JOIN tool_categories tc ON t.id = tc.tool_id` / total |
| 首次同步成功率 | ≥ 95%（新增+更新 / 总处理数） | `(tools_added + tools_updated) / (tools_added + tools_updated + errors)` 从 sync_logs |
| 快照持久化 | 同步后 tool_stats_history 新增行数 = tools 总数 | 每次同步后日志输出对比 |

### 4.2 后端性能指标

| 验收项 | 量化标准 | 测量方法 |
|--------|----------|----------|
| `/api/tools` 响应时间 | P95 < 300ms（1000 条数据） | locust/wrk 压测 100 并发 |
| `/api/tools/{slug}` 响应时间 | P95 < 100ms | 同上 |
| `/api/tools/compare?slugs=3个` | P95 < 200ms | 同上 |
| `/api/trends/rising` | P95 < 500ms | 同上（涉及时间序列计算） |
| 同步任务耗时（增量） | < 5 分钟（处理 ≤ 50 条变更） | sync_logs.finished_at - started_at |
| API 错误率 | < 1% | 日志统计 / 监控 |

### 4.3 前端指标

| 验收项 | 量化标准 | 测量方法 |
|--------|----------|----------|
| 首屏加载时间 | < 2s (Lighthouse) | Chrome DevTools Lighthouse |
| 搜索交互延迟 | 输入后 < 300ms 出结果（防抖后） | React DevTools Profiler |
| 对比页切换 | 添加/移除工具 < 200ms 更新 | 手动测试 |
| ECharts 渲染 | 趋势面板所有图表 < 1s 渲染 | 手动测试 |
| 响应式适配 | 1920px / 1366px / 768px 三档无布局错乱 | 手动截图对比 |
| 空状态/错误状态 | 搜索无结果/API 异常时有友好提示 | 手动测试 + 截图 |

### 4.4 功能完整性

| 验收项 | 量化标准 | 测量方法 |
|--------|----------|----------|
| 搜索功能 | 输入 "RNA" 返回含 RNA-seq/RNA-Seq/scRNA-seq 等工具 | 手动测试 ≥ 5 个搜索词 |
| 分类筛选 | 每个分类筛选后结果集不包含其他分类 | 手动抽查 |
| 排序 | stars desc / updated desc 排序结果单调 | 目视校验前 10 条 |
| 对比上限 | 第 5 个工具添加时被拒绝（前端拦截） | 手动测试 |
| 同步触发 | POST /api/sync/trigger 返回 202 + sync_logs 新记录 | curl + DB 查询 |
| OpenAPI 文档 | /docs 页面可访问且列出所有端点 | 浏览器访问 |

## 五、日志体系设计

### 5.1 日志层级

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| **DEBUG** | 开发调试细节 | SQL 参数、API 请求体 |
| **INFO** | 关键业务节点 | 同步开始/结束、API 请求到达、定时任务触发 |
| **WARNING** | 可恢复异常 | API 超时重试、数据字段缺失、去重冲突 |
| **ERROR** | 不可恢复错误 | 数据库连接失败、外部 API 连续失败、同步崩溃 |

### 5.4 日志输出配置

| 环境 | 输出目标 | 最低级别 |
|------|----------|----------|
| 开发 | console (彩色) | DEBUG |

### 5.5 sync_logs 数据库日志表

```sql
-- 每次同步写入一条记录，与 Python logging 互补
-- Python logging: 实时过程细节
-- sync_logs 表: 结构化汇总，可查询可聚合
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | - |
| source | VARCHAR(50) | github / biotools / all |
| trigger | VARCHAR(20) | manual / scheduled |
| status | VARCHAR(20) | success / partial / failed |
| tools_added | INT | 新增数 |
| tools_updated | INT | 更新数 |
| tools_skipped | INT | 跳过数 |
| tools_failed | INT | 失败数 |
| stats_snapshot_count | INT | 快照写入行数 |
| error_message | TEXT | 汇总错误信息 |
| duration_seconds | INT | 总耗时 |
| started_at | DATETIME(3) | 毫秒精度 |
| finished_at | DATETIME(3) | 毫秒精度 |

SQL 验收查询：

```sql
-- 每日同步覆盖率检查
SELECT DATE(started_at) as sync_date, source, status, 
       SUM(tools_added) as added, SUM(tools_updated) as updated, 
       AVG(duration_seconds) as avg_duration
FROM sync_logs 
WHERE started_at > DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY sync_date, source, status
ORDER BY sync_date DESC;

-- 同步失败率
SELECT source, 
       COUNT(CASE WHEN status = 'failed' THEN 1 END) * 100.0 / COUNT(*) as fail_rate_pct
FROM sync_logs 
WHERE started_at > DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY source;
```

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对策略 | 触发条件（日志告警） |
|------|------|------|----------|---------------------|
| GitHub API 限流 | 高 | 中 | Token 认证(5000 req/h) + 指数退避 | WARNING 日志中 rate_limit_remaining < 100 |
| bio.tools API 不稳定 | 中 | 中 | httpx 30s timeout + 3次重试 + 跳过继续 | WARNING 日志出现连续 3 次重试 |
| 数据去重失败 | 中 | 低 | 以 github_url 为唯一键，slug 为辅 | INFO 日志打印去重统计 |
| 同步数据量暴涨 | 低 | 中 | 首次全量后改为增量（按 updated 时间） | INFO 日志打印单次处理量，超阈值 WARNING |
| 中文搜索质量差 | 中 | 中 | 一期 LIKE+FULLTEXT，二期上 Elasticsearch | DEBUG 日志输出实际 SQL 和结果数 |
