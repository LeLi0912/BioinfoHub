## 二、数据库设计

### 2.1 表结构

#### tools

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK AUTO_INCREMENT | - |
| name | VARCHAR(255) | NOT NULL | 工具名称 |
| slug | VARCHAR(255) | UNIQUE NOT NULL | URL 友好标识 |
| description | TEXT | - | 工具描述(中文, 可后续翻译) |
| description_en | TEXT | - | 原始英文描述 |
| homepage_url | VARCHAR(1024) | - | 官网链接 |
| github_url | VARCHAR(1024) | UNIQUE | GitHub 仓库链接(去重键) |
| github_stars | INT | DEFAULT 0 | Star 数 |
| github_forks | INT | DEFAULT 0 | Fork 数 |
| github_open_issues | INT | DEFAULT 0 | 未关闭 Issues |
| github_license | VARCHAR(100) | - | 开源协议 |
| github_language | VARCHAR(50) | - | 主要编程语言 |
| github_topics | JSON | - | GitHub Topics 列表 |
| github_description | TEXT | - | GitHub 仓库描述 |
| github_updated_at | DATETIME | - | 最近代码更新 |
| github_created_at | DATETIME | - | 仓库创建时间 |
| biotools_id | VARCHAR(100) | - | bio.tools 对应 ID |
| publication_doi | VARCHAR(255) | - | 发表论文 DOI |
| publication_title | TEXT | - | 论文标题 |
| publication_year | INT | - | 发表年份 |
| citation_count | INT | DEFAULT 0 | 引用数 |
| is_active | BOOLEAN | DEFAULT TRUE | 是否仍在维护 |
| created_at | DATETIME | DEFAULT NOW() | 入库时间 |
| updated_at | DATETIME | DEFAULT NOW() ON UPDATE | 最后更新时间 |

索引：
- `UNIQUE INDEX idx_slug (slug)` — 详情页主入口
- `UNIQUE INDEX idx_github_url (github_url)` — 去重
- `INDEX idx_stars (github_stars)` — 热度排序
- `INDEX idx_updated (github_updated_at)` — 最近更新排序
- `INDEX idx_created (created_at)` — 最新入库排序
- `INDEX idx_language (github_language)` — 语言筛选
- `FULLTEXT INDEX idx_ft_name_desc (name, description, description_en)` — 全文搜索

#### categories

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK AUTO_INCREMENT | - |
| name | VARCHAR(100) | NOT NULL | 分类名(中文) |
| name_en | VARCHAR(100) | - | 分类名(英文) |
| slug | VARCHAR(100) | UNIQUE NOT NULL | URL 标识 |
| edam_id | VARCHAR(50) | - | EDAM ontology 对应术语 |

预设分类数据：
```
NGS/高通量测序, 单细胞组学, 蛋白质组学, 代谢组学,
结构生物学, 系统生物学, 影像组学, 表观遗传学,
转录组学, 基因组编辑, 药物发现, 其他
```

#### tool_categories

| 列 | 类型 | 约束 |
|---|---|---|
| tool_id | INT | FK → tools.id |
| category_id | INT | FK → categories.id |
| PRIMARY KEY (tool_id, category_id) | | |

#### tool_stats_history

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK AUTO_INCREMENT | - |
| tool_id | INT | FK → tools.id | - |
| stars | INT | - | 当日 Star 数 |
| forks | INT | - | 当日 Fork 数 |
| open_issues | INT | - | 当日 Issue 数 |
| snapshot_date | DATE | NOT NULL | 快照日期 |
| UNIQUE (tool_id, snapshot_date) | | | 每天每个工具一条 |

#### sync_logs

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK AUTO_INCREMENT | - |
| source | VARCHAR(50) | NOT NULL | github / biotools / all |
| trigger | VARCHAR(20) | NOT NULL | manual / scheduled |
| status | VARCHAR(20) | NOT NULL | success / partial / failed |
| tools_added | INT | DEFAULT 0 | 新增数 |
| tools_updated | INT | DEFAULT 0 | 更新数 |

| error_message | TEXT | - | 汇总错误信息 |

| started_at | DATETIME(3) | NOT NULL | 毫秒精度 |
| finished_at | DATETIME(3) | - | 毫秒精度 |

## 三、API 设计

### 3.1 工具 API

**GET `/api/tools`** — 工具列表
```
Query Params:
  q         : string   — 关键词搜索
  category  : string   — 分类 slug
  language  : string   — 编程语言
  sort      : string   — stars / updated / created (default: stars)
  order     : string   — asc / desc (default: desc)
  page      : int      — 页码 (default: 1)
  page_size : int      — 每页条数 (default: 20, max: 100)

Response 200:
{
  "items": [ ToolSummary, ... ],
  "total": 523,
  "page": 1,
  "page_size": 20,
  "pages": 27
}
```

**GET `/api/tools/hot`** — 热门排行
```
Query Params:
  limit : int — Top N (default: 20, max: 50)

Response 200:
{
  "items": [ ToolSummary, ... ]
}
```

**GET `/api/tools/{slug}`** — 工具详情
```
Response 200: { ...ToolDetail }
Response 404: { "detail": "工具不存在" }
```

### 3.2 对比 API

**GET `/api/tools/compare`** — 工具对比
```
Query Params:
  slugs : string — 逗号分隔, 最多 4 个 (例: star,samtools,gatk)

Response 200:
{
  "items": [ ToolCompareItem, ... ]
}
Response 400: { "detail": "最多只能对比 4 个工具" }
Response 404: { "detail": "工具 xxx 不存在" }
```

### 3.3 趋势 API

**GET `/api/trends/rising`** — 涨星最快
```
Query Params:
  days  : int — 统计天数 (default: 30, max: 90)
  limit : int — Top N (default: 20)

Response 200:
{
  "items": [
    {
      "tool_id": 1,
      "tool_name": "...",
      "tool_slug": "...",
      "stars_start": 1200,
      "stars_end": 3500,
      "stars_gained": 2300,
      "growth_rate": 76.7,
      "category_names": ["NGS"]
    }, ...
  ]
}
```

**GET `/api/trends/categories`** — 分类分布
```
Response 200:
[
  { "category_name": "NGS", "count": 180 },
  { "category_name": "单细胞组学", "count": 95 },
  ...
]
```

**GET `/api/trends/languages`** — 语言分布
```
Response 200:
[
  { "language": "Python", "count": 200 },
  { "language": "R", "count": 150 },
  ...
]
```

**GET `/api/trends/timeline`** — 新增趋势
```
Query Params:
  days : int — 统计天数 (default: 90, max: 365)

Response 200:
[
  { "date": "2026-04-01", "count": 3 },
  { "date": "2026-04-02", "count": 5 },
  ...
]
```

### 3.4 分类 API

**GET `/api/categories`**
```
Response 200:
[
  { "id": 1, "name": "NGS/高通量测序", "slug": "ngs", "count": 180 },
  ...
]
```

### 3.5 同步 API

**POST `/api/sync/trigger`**
```
Response 202:
{ "message": "同步任务已启动", "sync_log_id": 42 }
Response 409:
{ "detail": "同步任务正在进行中, sync_log_id=41" }
```

**GET `/api/sync/logs`**
```
Query Params:
  page / page_size

Response 200: 分页同步历史
```

## 四、实施步骤与验收标准

### Step 1: 项目初始化 [P]

**操作**：
1. 创建 `backend/` 目录结构
2. `poetry init` + 安装依赖 (fastapi, uvicorn, sqlalchemy[asyncio], aiomysql, alembic, httpx, apscheduler, pydantic)
3. 创建 `frontend/` + `npm create vite@latest` (React + TypeScript)
4. 安装前端依赖 (antd, @ant-design/icons, echarts, echarts-for-react, axios, react-router-dom)
5. 配置 `backend/app/config.py` 和 `backend/app/logger.py`

**验收标准**：
- [ ] `poetry run uvicorn app.main:app` 启动成功，访问 `http://localhost:8000/docs` 可看到 Swagger 页面
- [ ] `npm run dev` 启动成功，访问 `http://localhost:5173` 可看到默认 React 页面
- [ ] 日志配置生效：启动时控制台输出 `[INFO] bioinfohub.main: 应用启动, env=development`

### Step 4: 数据聚合服务（同步逻辑核心）

**操作**：
1. `backend/app/services/github.py`:
   - `search_repos(keywords, page, per_page)` — 调用 `GET /search/repositories?q=topic:bioinformatics+...`
   - `get_repo(owner, repo)` — 获取单个仓库详情
   - `extract_tool_data(repo_json)` — JSON → dict 映射
   - Rate limit 处理：指数退避 1s/2s/4s/8s，最多 3 次重试
2. `backend/app/services/biotools.py`:
   - `fetch_tools(page)` — 调用 bio.tools API
   - `map_edam_to_category(edam_terms)` — EDAM 术语 → 本地分类映射
3. `backend/app/services/sync.py`:
   - `run_sync(source, trigger)` — 编排同步流程
   - 去重逻辑：先按 github_url 匹配 → 再按 slug 匹配 → 都不匹配则 INSERT
   - 写入 tool_stats_history 快照

**验收标准**：
- [ ] 手动调用 `run_sync("github", "manual")` 可从 GitHub 拉取到真实数据并写入 tools 表
- [ ] 同一 github_url 的工具重复执行同步不会产生重复行（UPDATE 而非 INSERT）
- [ ] sync_logs 记录 status / tools_added / tools_updated / duration_seconds 准确
- [ ] GitHub rate limit 超限时日志输出 WARNING 并重试
- [ ] bio.tools 超时时日志输出 WARNING 并跳过（不崩溃）

**日志输出要求**：

### Step 6: 路由层 — 对比与趋势

**操作**：
1. `backend/app/routers/compare.py`: `GET /api/tools/compare?slugs=a,b,c`
2. `backend/app/routers/trends.py`:
   - `GET /api/trends/rising` — 30 天增量斜率计算
   - `GET /api/trends/categories` — 分组计数
   - `GET /api/trends/languages` — 分组计数
   - `GET /api/trends/timeline` — 按日期分组的新增工具数
3. `backend/app/services/trends.py` — 趋势计算逻辑(子查询 + 窗口函数或 Python 内存计算)

**验收标准**：
- [ ] `GET /api/tools/compare?slugs=star,samtools,gatk` 返回 3 个工具的并行对比数据
- [ ] `GET /api/tools/compare?slugs=a,b,c,d,e` (5 个) 返回 400 "最多只能对比 4 个"
- [ ] `GET /api/trends/rising?days=30` 返回按增量排序的列表，增长率为正值
- [ ] `GET /api/trends/categories` 各分类 count 之和 = tools 总数
- [ ] `GET /api/trends/languages` 各语言 count 之和 ≤ tools 总数(允许 NULL 语言)
- [ ] `GET /api/trends/timeline?days=90` 返回 90 个日期点

- 分类/语言无对应工具时返回空数组而非 null

### Step 7: 定时任务调度 + 同步 API

**操作**：
1. `backend/app/tasks/scheduler.py`:
   - FastAPI `lifespan` 中启动 APScheduler
   - 注册 cron job: `0 2 * * *` 执行 `run_sync("all", "scheduled")`
   - 防重入：检查最近一条 sync_log 是否 status='running'，是则跳过
2. `backend/app/routers/sync.py`:
   - `POST /api/sync/trigger` — 手动触发，返回 202 + sync_log_id
   - `GET /api/sync/logs` — 分页返回同步历史

**验收标准**：

## 八、日志配置实现要点

### Python (loguru)

```python
# backend/app/logger.py — 概要(代码实现时不写注释)
import sys
from loguru import logger
from pathlib import Path

def setup_logger(env: str = "development"):
    logger.remove()

    # 开发环境: 彩色 console, DEBUG 级别
    if env == "development":
        logger.add(sys.stderr, level="DEBUG", colorize=True,
                   format="<green>{time:HH:mm:ss}</green> | <level>{level: <7}</level> | <cyan>{name}</cyan> | <level>{message}</level>")

    # 生产环境: console INFO + 文件轮转
    if env == "production":
        logger.add(sys.stderr, level="INFO",
                   format="{time:YYYY-MM-DD HH:mm:ss} | {level: <7} | {name} | {message}")
        logger.add("logs/app_{time:YYYY-MM-DD}.log", rotation="10 MB", retention="30 days",
                   level="INFO", encoding="utf-8")
        logger.add("logs/sync_{time:YYYY-MM-DD}.log", rotation="50 MB", retention="60 days",
                   level="INFO", encoding="utf-8", filter=lambda r: r["name"].startswith("bioinfohub.sync"))

    return logger
```

### 前端 (console wrapper)

```typescript
// frontend/src/api/logger.ts — 概要
const PREFIX = "[BioinfoHub]";

export const logger = {
```
